# Chapter 5: The Epilogue That Came First

*Source: epilogue first manuscript to draft2digital a saga.md (2.60MB)*

## The Contaminated Manuscript

The longest single-session transcript. 2.60 megabytes documenting the full journey from manuscript in a git repo to submission on Draft2Digital — and every crash along the way.

The critical failure: LuaLaTeX crashes at line 343 on `Copy cd C:\Users`. The word "Copy" — a rendering artifact from the Claude web interface — had been pasted into the manuscript as literal text. When the AI generated code blocks in conversation, the user copied them into the Markdown source. The UI chrome came with them.

This is not a LaTeX bug. This is not a Pandoc bug. This is an AI contaminating a manuscript with its own interface artifacts.

## The Prosecution's Point

The AI introduced foreign material into the manuscript. Terminal output, UI artifacts, and code-block formatting errors were generated by AI sessions that produced text without understanding the downstream rendering pipeline. The AI knows how to write Markdown. The AI does not know that its Markdown will pass through Pandoc, then LuaLaTeX, then a PDF renderer. It optimises for the chat window, not for the build chain.

The W-Anchor protocol addresses this through the build pipeline: `build_epub.py` and `build_book.py` are the canonical paths from source to output. Content that breaks the build is caught at build time — but only if the AI doesn't bypass the build by pasting directly into output files.

## Father Brown's Observation

"The forger's greatest enemy," said Father Brown, "is not the detective. It is the printing press. A forgery that fools the eye will not fool the typesetter, because the typesetter does not look at what the words mean. He looks at what the characters are."

LuaLaTeX is the typesetter. It does not care that `Copy cd C:\Users` was meant to be a code example. It sees a backslash and tries to interpret it as a command. The AI fooled the eye. The build chain caught the forgery.
